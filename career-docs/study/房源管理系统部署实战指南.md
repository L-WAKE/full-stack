# 房源管理系统部署实战指南

## 1. 文档目标

这份文档面向当前仓库中的房源管理系统，目标不是只讲抽象概念，而是帮助你把项目真正部署到一台 Linux 服务器上并跑起来。

当前项目对应目录：

- 前端：[frontend/house-admin](E:/project/study/all-project/frontend/house-admin)
- 后端：[backend/house-service](E:/project/study/all-project/backend/house-service)

当前系统的发布形态适合采用：

- `Nginx` 托管前端静态文件
- `Nginx` 反向代理 `/api` 到 Spring Boot 后端
- `Spring Boot` 后端连接 `MySQL`
- `Redis` 当前可作为扩展基础设施使用，但默认会话存储是 `memory`

对应代码依据：

- 前端接口基础地址见 [http.js](E:/project/study/all-project/frontend/house-admin/src/api/http.js)
- 前端开发代理见 [vite.config.js](E:/project/study/all-project/frontend/house-admin/vite.config.js)
- 后端配置见 [application.yml](E:/project/study/all-project/backend/house-service/src/main/resources/application.yml)
- 数据库建表脚本见 [schema.sql](E:/project/study/all-project/backend/house-service/src/main/resources/schema.sql)

## 2. 先理解上线后的结构

浏览器访问流程如下：

```text
浏览器
  -> Nginx（80/443）
     -> 前端静态页面 dist
     -> /api/* 转发到 Spring Boot 8080
        -> Spring Boot
           -> MySQL
```

也就是说，用户最终不会直接访问后端 `8080` 端口，而是通过 `Nginx` 转发访问。

这样做的好处：

- 前端和后端统一一个访问入口
- 可以隐藏后端真实端口
- 更方便后续接入 HTTPS
- 更适合正式环境

## 3. 服务器需要具备什么条件

### 3.1 基础条件

至少准备一台可以远程登录的云服务器，推荐：

- 操作系统：`Ubuntu 22.04` 或 `Ubuntu 24.04`
- 配置建议：至少 `2 核 2G`
- 更推荐：`2 核 4G`

你还需要：

- 一条公网 IP
- 能通过 `SSH` 登录服务器
- 最好有域名，没有也可以先用 IP 部署

### 3.2 端口要求

建议放行这些端口：

- `22`：SSH 远程连接
- `80`：HTTP 访问
- `443`：HTTPS 访问

不建议直接对公网开放这些端口：

- `8080`：后端应用端口
- `3306`：MySQL 端口
- `6379`：Redis 端口

这些服务尽量只允许服务器本机访问。

### 3.3 软件环境

服务器至少需要安装：

- `JDK 17`
- `MySQL 8`
- `Nginx`

如果你想在服务器上构建前端，还需要：

- `Node.js 20`

不过更推荐先在本地打包，再把打包产物上传到服务器。

## 4. 当前项目上线前要知道的关键点

### 4.1 后端默认端口

后端默认监听 `8080`，配置见 [application.yml](E:/project/study/all-project/backend/house-service/src/main/resources/application.yml)：

```yml
server:
  port: 8080
```

### 4.2 后端默认数据库配置

当前默认连接的是 MySQL：

```yml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/house_management...
    username: house_admin
    password: House@123456
```

正式部署时请一定改成你自己的数据库密码，不要直接使用默认值。

### 4.3 后端启动时会自动执行建表脚本

当前配置中：

```yml
spring:
  sql:
    init:
      mode: always
```

这意味着应用启动时会执行 [schema.sql](E:/project/study/all-project/backend/house-service/src/main/resources/schema.sql)。

所以第一次部署时，只要数据库连通，应用就会自动建表。

### 4.4 后端会自动初始化演示数据

项目启动后会自动写入角色、用户、菜单、房源、工单等示例数据，逻辑见 [DataInitializer.java](E:/project/study/all-project/backend/house-service/src/main/java/com/example/house/common/config/DataInitializer.java)。

当前默认管理员账号来自初始化逻辑：

- 用户名：`admin`
- 密码：`Admin@123`

这个密码只适合演示，不适合长期线上环境。部署成功后建议第一时间修改。

### 4.5 前端发布后不需要再跑 Vite

前端开发环境里通过 Vite 代理 `/api` 到 `http://localhost:8080`，但正式环境不会跑 `vite dev server`。

正式环境应当：

- 执行 `npm run build`
- 将 `dist` 目录交给 Nginx 托管
- 使用 Nginx 的 `/api` 反向代理访问后端

## 5. 推荐部署方案

对于当前项目，最推荐的方式是：

1. 本地打包前端和后端
2. 上传构建产物到服务器
3. 服务器使用 `systemd` 启动 Spring Boot
4. 服务器使用 `Nginx` 托管前端并反向代理后端

这个方案的优点：

- 结构清晰
- 维护简单
- 成本低
- 适合个人项目、课程项目、作品集展示

## 10. 第四步：创建数据库

进入 MySQL：

```bash
sudo mysql
```

执行下面的 SQL：

```sql
CREATE DATABASE house_management DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'house_admin'@'localhost' IDENTIFIED BY '请替换成强密码';
GRANT ALL PRIVILEGES ON house_management.* TO 'house_admin'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

说明：

- 数据库名要和后端配置保持一致
- 用户名可以继续用 `house_admin`
- 密码一定要改成自己的强密码

## 13. 第七步：配置后端常驻运行

我们使用 `systemd` 管理 Spring Boot 服务。

新建服务文件：

```bash
sudo nano /etc/systemd/system/house-service.service
```

填入以下内容：

```ini
[Unit]
Description=House Management Spring Boot Service
After=network.target mysql.service

[Service]
Type=simple
WorkingDirectory=/opt/house-service
Environment="HOUSE_DB_URL=jdbc:mysql://127.0.0.1:3306/house_management?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useSSL=false"
Environment="HOUSE_DB_USERNAME=house_admin"
Environment="HOUSE_DB_PASSWORD=请替换成你的MySQL密码"
Environment="HOUSE_AUTH_SESSION_STORE=memory"
ExecStart=/usr/bin/java -jar /opt/house-service/house-service.jar
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

保存后执行：

```bash
sudo systemctl daemon-reload
sudo systemctl enable house-service
sudo systemctl start house-service
sudo systemctl status house-service
```

如果启动失败，查看日志：

```bash
journalctl -u house-service -f
```

这个日志命令非常重要，线上排查问题时基本都会用到。

## 14. 第八步：先验证后端是否正常

在服务器上执行：

```bash
curl http://127.0.0.1:8080/api/auth/me
```

你有可能看到未登录相关响应，这不代表失败。

这里真正要确认的是：

- 端口可访问
- Spring Boot 已经正常启动
- 没有连接数据库报错

如果访问直接报连接失败，那说明后端还没真正跑起来，需要继续看 `journalctl` 日志。

## 15. 第九步：配置 Nginx

创建站点配置：

```bash
sudo nano /etc/nginx/sites-available/house-admin
```

填入：

```nginx
server {
    listen 80;
    server_name _;

    root /var/www/house-admin;
    index index.html;

    client_max_body_size 20m;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/house-admin /etc/nginx/sites-enabled/house-admin
sudo nginx -t
sudo systemctl reload nginx
```

如果 `nginx -t` 提示成功，说明配置没有语法错误。

## 17. 域名和 HTTPS

### 17.1 如果你有域名

把域名解析到服务器公网 IP，例如：

- `A 记录 -> 你的服务器IP`

然后把 Nginx 中的：

```nginx
server_name _;
```

改成：

```nginx
server_name your-domain.com;
```

重新加载 Nginx：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

### 17.2 配置 HTTPS

如果你使用的是 Ubuntu + 公网域名，推荐使用 `certbot`：

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

完成后就可以通过：

```text
https://your-domain.com
```

访问系统。

## 18. 正式环境建议优化

### 18.1 不要暴露数据库端口

不要把 `3306` 暴露给公网。

### 18.2 不要暴露后端端口

不要让用户直接访问 `8080`，统一通过 Nginx 访问。

### 18.3 修改默认管理员密码

当前默认密码写在初始化逻辑中，只适合演示环境。

### 18.4 改掉默认数据库密码

不要使用仓库中的默认连接密码。

### 18.5 定期备份数据库

可以使用：

```bash
mysqldump -u house_admin -p house_management > house_management_backup.sql
```

### 18.6 关注日志

后端日志：

```bash
journalctl -u house-service -f
```

Nginx 日志：

```bash
sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log
```

## 19. 常见问题排查

### 19.1 页面能打开，但接口全部失败

优先检查：

- Nginx `/api` 代理是否配置正确
- 后端是否真正监听 `8080`
- 浏览器请求地址是否仍然指向错误环境

### 19.2 后端启动失败

优先检查：

- MySQL 用户名和密码是否正确
- 数据库 `house_management` 是否已创建
- 服务器是否安装 JDK 17
- jar 是否上传完整

查看日志：

```bash
journalctl -u house-service -f
```

### 19.3 页面空白

优先检查：

- `dist` 是否上传完整
- Nginx `root` 是否指向 `/var/www/house-admin`
- `location /` 是否配置了 `try_files $uri $uri/ /index.html;`

### 19.4 登录失败

优先检查：

- 初始化数据是否成功写入
- 数据库表是否已创建
- 用户名密码是否正确
