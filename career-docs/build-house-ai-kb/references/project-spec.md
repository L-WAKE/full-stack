# 项目说明

## 项目范围

构建一个面向房源管理系统的 AI 知识库问答平台。
第一阶段重点是做出稳定可用的 RAG 知识库，而不是一开始就上复杂自治 Agent。

首期必须支持：

- 登录
- 知识库空间管理
- PDF 和 DOCX 上传
- 文档解析
- 文档切片
- 向量化与索引
- 检索问答
- 引用来源展示
- 会话管理
- SSE 流式输出
- 权限隔离

二期可以逐步扩展：

- 工具调用
- 业务 API 联动
- 智能表单填充
- 流程引导
- 工单助手
- SQL 查询助手

## 产品目标

- 回答“某个功能怎么用”“某个菜单在哪里”“某个业务流程怎么走”等问题。
- 将现有业务文档转化为可检索、可引用、可对话的 AI 能力。
- 体现完整的 AI 工程能力：前端、后端、数据、检索、流式输出、权限、部署。
- 保证这个项目在简历和面试中容易讲清楚，因为主链路完整、真实、可验证。

## 推荐技术栈

前端：

- Vue 3
- JavaScript
- Vite
- Pinia
- Vue Router
- Element Plus
- Axios

后端：

- Python 3.11
- FastAPI
- Pydantic
- SQLAlchemy
- Alembic
- Celery 或 FastAPI BackgroundTasks

数据和基础设施：

- PostgreSQL
- Redis
- Qdrant
- 本地文件存储
- Docker / Docker Compose

模型层：

- Chat、Embedding、Rerank 三类能力分开封装
- 优先保留 OpenAI 兼容接口接入能力，也为阿里百炼、硅基流动、DeepSeek、Qwen 等平台预留空间

## 架构思路

高层链路：

`Vue3 前端 -> Nginx -> FastAPI -> PostgreSQL / Redis / Qdrant / 文件存储 / 大模型服务`

Worker 负责：

- 文档解析
- 文本切片
- 向量化
- 索引构建或重建

系统职责划分：

- 前端：知识库管理、上传页面、聊天页面、引用展示
- FastAPI：鉴权、文档接口、检索接口、会话接口、流式输出接口
- PostgreSQL：用户、空间、文档、切片、会话、引用等结构化数据
- Redis：队列、缓存、上下文加速
- Qdrant：向量存储和检索

## 知识来源建议

优先使用第一个房源管理系统的真实资料：

- 需求文档
- 菜单说明文档
- 操作手册
- FAQ
- 培训资料
- 接口文档
- 维修流程和保洁流程文档

## 核心产品模块

鉴权与权限：

- 用户登录
- 当前用户信息
- 可访问知识库空间
- 按空间、文档、会话进行隔离

知识库空间：

- 创建和编辑空间
- 分配成员
- 查看文档数量和切片数量

文档中心：

- 上传文件
- 校验文件类型和大小
- 记录文档元数据
- 展示解析状态

文档解析：

- 从 PDF 和 DOCX 提取文本
- 尽量保留标题与段落结构
- 保留页码和分段信息

知识切片：

- 优先保留语义完整性
- 先按标题，再按段落，再按长度二次切分
- 保留合理重叠

检索问答：

- 混合检索
- 可选重排
- 上下文拼装
- 基于检索结果生成带引用回答

会话系统：

- 创建、重命名、删除、查看历史
- 存储消息和引用
- 会话必须绑定知识库空间

## 质量标准

一个合格的首期版本，不要求花哨 Agent 能力。
但必须做到：

- 能回答
- 能引用
- 权限正确
- 链路完整
- 结构清晰
- 本地可部署
